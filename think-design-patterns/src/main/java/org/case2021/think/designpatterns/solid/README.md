#开发原则
- 描述
```
SOLID原则：由 5 个设计原则组成的，它们分别是：单一职责原则、开闭原则、里式替换原则、接口隔离原则和依赖反转原则，
依次对应 SOLID 中的 S、O、L、I、D 这 5 个英文字母，
SRP单一职责原则 Single Responsibility Principle；
KISS保持简单 Keep It Simple and Stupid； 
YAGNI不需要原则 You Ain’t Gonna Need It ；
DRY 不要重复原则 Don’t Repeat Yourself ； 
LOD 迪米特法则 Law of Demeter
```
- 理解
```
"看懂"和"会用"这是两码事；"用好"更是难上加难；不能拿原则当真理，要把握重点，做到活学活用
```
